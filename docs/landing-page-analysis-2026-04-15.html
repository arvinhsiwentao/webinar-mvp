🔍 為什麼指向 UI 而非投放品質？

6 倍差距有三種可能解釋：

可能原因說明判讀
A. 投放品質差 Google Ads 把低意圖流量塞給手機 可能性低
B. 手機用戶天生意圖較低 產業通識：高決策門檻產品，手機轉換天生比桌機低 1.5-3 倍 一定有（但無法解釋全部）
C. 手機 UI 體驗差 用戶有意圖但被 UI 擋下來 最可能的主因

先解釋一個關鍵詞：「UI 發現/點擊」是什麼

= 用戶「看見 CTA 並按下去」這個動作本身（對應漏斗的「PV → 點時段卡」）。拆成兩件事：

關鍵字意思失敗的情況
發現 (visibility) 用戶有沒有注意到 CTA 存在 CTA 在 fold 以下、被大圖蓋住、動畫太多分散注意力、看起來不像按鈕
點擊 (tapability) 有沒有能力順利點下去 按鈕太小、被 sticky 元件擋住、誤觸相鄰元素、手機鍵盤遮住、載入延遲

兩件事都是 UI 設計決定的，跟用戶「要不要買」的意圖無關 — 這是為什麼差距集中在這一步特別指向 UI。

把漏斗三個階段依性質分類：

階段性質主要受什麼影響
PV → 點時段卡UI 發現/點擊視覺設計、按鈕可見性、頁面長度
開 modal → form_start有意願開始填過半是用戶意圖，少部分是 modal 設計
form_start → sign_up有意願填完送出主要是意圖 + 表單欄位友善度
一句話總結：手機用戶如果能走到 form_start，他們的完成率跟桌機一樣好甚至更好（70% vs 62%）。問題是絕大多數手機用戶連「找到並按 CTA」這關都過不去。

三個支持 C（而非 A）的證據

  1. 手機 CPC 很便宜（NT$8） — Google Ads quality score 機制會把品質差的流量 CPC 拉高。手機 CPC 這麼低，代表至少不是 Google 認為品質很差的那批。
  2. 差距位置集中在「UI 發現/點擊」階段：
    漏斗步驟MobileDesktop差距倍數意涵
    PV → 點時段卡 1.4% 6.5% 4.6x UI 影響最大 — 差距集中在這
    開 modal → form_start 77% 103%* 1.3x 小幅差距
    form_start → sign_up 70% 62% — 手機反而略高

    * Desktop 103% = 有用戶沒走完整 modal 流程但觸發 form_start（e.g. 直接從 schedule anchor 進）。
    如果是純意圖差異，差距應該全漏斗均勻。但實際上 差距高度集中在「CTA 發現/點擊」階段，這正是 UI 影響最大的地方。

  3. 程式碼層具體可見的 UI 缺陷 — Hero 大圖佔滿手機 viewport、CTA 在 fold 之下；sticky bar 文案太被動；多個動畫同時跑分散注意力。詳見下方「📱 手機 UI 具體 friction 點」一節。

更謹慎的結論表述

6.1 倍差距裡，產業基線能解釋 1.5-3 倍（手機意圖天生較低），剩下的 2-3 倍超額差距主要歸咎於手機 UI。

反過來說：即使悲觀假設手機意圖只有桌機的一半，現在也應該拿到 2-3% 的 conversion，而不是 0.93%。中間差的那 1.5 倍以上，就是手機 UI 可以拿回來的機會。

關鍵洞察：手機問題在「點 CTA → 開 modal」之前

  • 手機只有 1.4% PV 點到 CTA / 桌機 6.5% — 手機用戶根本沒被頁面說服按下任何按鈕，差距 4.6x
  • 手機開了 modal 又有 23% 不填表（vs 桌機 form_start > 開啟，幾乎全填）— 可能 modal 在手機上太擠或 first impression 不友好
  • 填了之後完成率（form_start → sign_up）兩邊接近 — 表單本身在手機上 OK

可能的手機 friction 源（待逐項驗證）

  • Hero 大圖在手機上佔據絕大部分視覺，CTA 按鈕被擠下去
  • Sticky bar 文案不夠「行動感」或被瀏覽器底部 UI 擋住
  • Schedule 卡在手機上難點（觸控目標太小？卡片太多需要捲動？）
  • Modal 開啟後手機鍵盤彈出佔半屏，用戶看不到所有欄位
  • 整體頁面長度過長，手機滑到 schedule 卡之前已疲勞離開

關鍵洞察

反事實計算（What If）

  • 同樣 NT$197,841 預算 → 註冊從 47 拉到 ~250 個
  • 手機 CPA 從 NT$4,209 → NT$791（降 81%）

🔑 這個 sample 真正能學到的 5 件事

  1. 完整看完直播 = 購買的強指標：91 signed_up → 27 完整看完直播 → 1 購買。看完組購買率 1/27 = 3.7%，沒看完的 ≈ 0%。對看完組追擊（email/push）ROI 最高。
  2. 這位用戶超快做決定：landing → signup 僅 105 秒、完整看完直播、直播中直接點 CTA 進 checkout。高意圖用戶的路徑很線性 — 不在頁面上來回游移。這表示頁面上為「猶豫用戶」多放的說服元件（證言、FAQ、多個 CTA），對最能轉換的高意圖族群可能反而是噪音。
  3. 在 checkout 頁考慮 18 分鐘，且在付款當下焦慮尋求支援：即使是高意圖用戶，付款那一刻還是會焦慮想找人確認。Chatbot + WhatsApp 在這個時點被用到了 — 不是決策觸發，但是 reassurance 機制。沒這個管道可能會讓焦慮用戶中途放棄。
  4. 手機能轉換，關鍵是體驗流暢：這位用戶是 Samsung 中低階 Android 手機，卻 56 分鐘內完成整條漏斗。問題不是「手機使用者不會買」，是「大多數手機使用者卡在前面 UI 那一步」。
  5. Campaign 2603_mike_dgc_A1 值得研究：這組素材帶來了唯一購買者（還 2 次回訪）。可到 ads_agent 表看這個 campaign 的 CTR / VScore / CPA，跟其他素材對比找成功模式。

位置問題嚴重度

填表階段可能的 friction（待 field-level event 驗證）

  COUNT(*) AS events

  AND event_name = 'c_scroll_depth'

  COUNTIF(event_name='sign_up') AS sign_up

  AND device.category IN ('mobile','desktop','tablet')

SELECT
  segments_device,
  SUM(metrics_cost_micros)/1e6 AS cost_twd,

  SAFE_DIVIDE(SUM(metrics_cost_micros)/1e6, SUM(metrics_clicks)) AS cpc
FROM `overseas-author.authors_google_ads_dataset.ads_AdBasicStats_4581920893`
WHERE _DATA_DATE BETWEEN ... AND ...

User-level 交叉驗證查詢（避開跨裝置歸因） 
WITH user_device AS (
  SELECT
    user_pseudo_id,
    ARRAY_AGG(device.category ORDER BY event_timestamp LIMIT 1)[OFFSET(0)] AS first_device,
    MAX(CASE WHEN event_name='page_view' THEN 1 ELSE 0 END) AS had_pv,
    MAX(CASE WHEN event_name='sign_up' THEN 1 ELSE 0 END) AS signed_up
  FROM `overseas-author.analytics_527332244.events_*`
  WHERE _TABLE_SUFFIX BETWEEN ... AND ...
  GROUP BY user_pseudo_id
)
SELECT first_device, COUNT(*) AS users,
  SUM(signed_up) AS signed_up_users,
  ROUND(100*SUM(signed_up)/COUNT(*), 2) AS conversion_pct
FROM user_device
WHERE had_pv = 1
GROUP BY first_device

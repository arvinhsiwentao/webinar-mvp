# Mux（影片託管 + 串流播放服務）

## 它是什麼

Mux 是一個「影片管家」—— 你把影片交給它，它幫你保管、整理、然後播放給全世界的觀眾看。就像 YouTube 幫你存影片和播放影片一樣，但 Mux 是給開發者用的幕後服務，觀眾看不到 Mux 的品牌，只會看到我們自己的網站。

本專案用 Mux 做三件事：讓管理員上傳預錄好的影片、自動把影片轉成適合網路播放的格式、然後在直播間頁面串流播放給觀眾。

## 為什麼選它

- **直接上傳（Direct Upload）** — 影片從管理員的電腦直接傳到 Mux 的伺服器，不需要先經過我們自己的伺服器。就像寄包裹時，不用先送到公司再轉寄，而是直接從你家寄到目的地，省時省力。而且支援「斷點續傳」（網路斷了也不用從頭來，會接著傳）
- **HLS 串流（自適應畫質播放）** — 觀眾網路好的時候看高畫質，網路差的時候自動降畫質，就像 Netflix 會根據你的網速調整畫面品質一樣，確保不會一直轉圈圈
- **CDN 全球分發** — Mux 在世界各地都有伺服器副本（CDN，內容分發網路），觀眾不管在哪裡，都能從離自己最近的伺服器拿到影片，速度更快。不需要我們自己搭建這套分發系統
- **簡單好用** — 影片上傳完，Mux 自動幫你轉好格式，給你一個播放網址就能用了。不用自己搞影片轉檔那些複雜的事

## 在專案中做什麼

Mux 在我們的專案裡負責三件事：

1. **影片上傳** — 管理員在後台上傳影片 → Mux 自動轉檔（把原始影片轉成適合網路串流的格式）→ 生成一個可以播放的網址
2. **影片播放** — 直播間頁面用 Video.js（一個影片播放器工具）讀取 Mux 提供的影片網址，播放給觀眾看
3. **影片管理** — 管理員可以在後台查看所有已上傳的影片、刪除不要的影片、或是把之前已經在 Mux 帳戶裡的影片直接匯入使用

## 需要設定的密碼和金鑰

Mux 就像一個有門禁的倉庫，要有通行證才能進去存放和取用影片。這些「通行證」叫做環境變數（Environment Variables，存在伺服器上的機密設定值，程式啟動時會讀取）：

| 通行證名稱 | 做什麼用的 |
|------|------|
| `MUX_TOKEN_ID` | Mux 帳號的身份證號碼，用來告訴 Mux「我是誰」 |
| `MUX_TOKEN_SECRET` | 配合身份證的密碼，兩個合在一起才能操作 Mux 的所有功能。絕對不能外洩 |

### 用到的程式套件

| 套件名稱 | 做什麼用的 |
|------|------|
| `@mux/mux-node` | 伺服器端的 Mux 工具包，負責跟 Mux 的伺服器溝通（建立上傳、查詢狀態、刪除影片等） |
| `@mux/upchunk` | 瀏覽器端的「搬運工」—— 把大檔案切成小塊一塊一塊上傳，網路斷了會自動重試，不用從頭來 |

## 資料怎麼流動

### 上傳流程（管理員上傳一支影片）

想像你要把一部電影寄到國外的倉庫保管：

```
【第一步：跟倉庫說「我要寄東西」】
管理員在後台選擇一個影片檔案
  ↓
我們的伺服器跟 Mux 說：「有人要上傳影片，請準備一個收件地址」
（POST /api/admin/videos → Mux 回傳一個專屬的上傳網址）
  ↓

【第二步：把影片切成小包裹寄出去】
UpChunk（瀏覽器端的搬運工）把影片檔案切成小塊，直接傳到 Mux

  檔案越大，每塊切得越大（減少寄送次數）：
  ├── 1GB 以下的影片 → 每塊 5MB
  ├── 1-5GB 的影片 → 每塊 16MB
  ├── 5-10GB 的影片 → 每塊 32MB
  └── 10GB 以上的影片 → 每塊 64MB

  如果某一塊傳失敗了，會自動重試（每塊最多重試 10 次，每次等 3 秒再試）
  如果網路完全斷線，搬運工會自動暫停，等網路恢復後接著傳
  ↓

【第三步：等倉庫整理貨物】
上傳完成後，Mux 開始「轉檔」（把影片轉成各種畫質，方便不同網速的觀眾播放）
我們的前端每 3 秒去問一次：「整理好了嗎？」
（GET /api/admin/videos/[id]/status，最多問 2 小時）
  ↓

【第四步：拿到播放網址】
Mux 轉檔完成 → 給我們一個 playback ID（播放用的識別碼）
→ 組成播放網址：https://stream.mux.com/{playbackId}.m3u8
→ 存進資料庫，狀態標記為「可用」
```

### 播放流程（觀眾在直播間看影片）

```
觀眾進入直播間 → 系統從資料庫拿到這場研討會對應的影片播放網址
  ↓
Video.js（影片播放器）看到網址結尾是 .m3u8
→ 知道要用 HLS 協議（一種專門為網路串流設計的播放方式）來播放
  ↓
播放網址格式：https://stream.mux.com/{playbackId}.m3u8
→ 影片從 Mux 的 CDN 串流到觀眾的瀏覽器
→ 自動根據觀眾的網速選擇合適的畫質
```

### 匯入已有的 Mux 影片

如果之前已經在 Mux 帳戶裡上傳過影片，不用重新傳一次，可以直接匯入使用：

```
管理員在後台按「從 Mux 導入」
  ↓
系統去 Mux 帳戶裡撈出所有已經轉好檔、可以公開播放的影片清單
（GET /api/admin/videos/mux-assets）
  ↓
管理員選一支 → 系統把這支影片的資訊存進我們的資料庫，標記為「可用」
（POST /api/admin/videos/import）
```

## 影片的三個階段

就像網購的物流狀態，每支影片會經歷三個階段：

```
uploading（上傳中）— 檔案正在從管理員的電腦傳到 Mux
  ↓ 上傳完成
processing（轉檔中）— Mux 正在把影片轉成適合串流的格式
  ↓ 轉檔完成
ready（可以播放）— 一切就緒，觀眾可以看了

如果中途出了問題：
processing → error（轉檔失敗）
```

## 相關程式檔案

| 檔案位置 | 負責什麼 |
|------|------|
| `src/lib/mux.ts` | Mux 的核心工具箱 — 初始化連線、建立上傳、查詢狀態、取得播放網址、列出所有影片、刪除影片 |
| `src/lib/video-upload.ts` | 前端的上傳指揮官 — 負責協調「要上傳網址 → 切塊上傳 → 等轉檔完成」整個流程 |
| `src/app/api/admin/videos/route.ts` | 影片 API 入口 — POST 建立新上傳 / GET 取得影片清單 |
| `src/app/api/admin/videos/[id]/status/route.ts` | 查詢某支影片的上傳或轉檔進度 |
| `src/app/api/admin/videos/[id]/route.ts` | 修改或刪除某支影片 |
| `src/app/api/admin/videos/import/route.ts` | 匯入 Mux 帳戶中已有的影片 |
| `src/app/api/admin/videos/mux-assets/route.ts` | 取得 Mux 帳戶中所有可用影片的清單 |
| `src/app/(admin)/admin/_components/VideoManager.tsx` | 管理後台的影片管理介面（管理員看到的操作畫面） |

## 需要注意的重要事項

1. **不用 Webhook，用輪詢** — Webhook 是「Mux 主動通知我們」的機制，但我們沒有用。我們選擇用「主動去問」的方式（每 3 秒問一次，最多問 2 小時）。為什麼？因為對這個專案的規模來說，輪詢更簡單、更容易除錯，不需要額外設定接收通知的端點

2. **所有影片都是公開存取** — 影片在 Mux 上設為 public（公開播放政策），任何人知道播放網址就能看。不需要額外的簽名驗證。這是因為我們的影片本來就是要給觀眾看的，不需要額外的存取限制

3. **斷點續傳自動處理** — UpChunk 搬運工會自動處理網路不穩的情況：斷線時暫停、恢復時接著傳、每一小塊最多重試 10 次（每次等 3 秒再試）。管理員不需要擔心上傳到一半斷網要重來

4. **刪除保護** — 如果某支影片正被某場研討會使用中，系統不允許刪除。避免不小心刪掉正在用的影片，導致直播間播不出東西

5. **影片大小上限** — 前端限制最大 20GB。超過的檔案需要先壓縮再上傳

6. **影片畫質設為 basic** — 上傳時 Mux 的轉檔品質設為 `basic`（基本畫質）。這是最經濟的方案，對研討會類型的影片內容已經足夠

7. **上傳網址 24 小時有效** — 每次建立上傳時，Mux 給的上傳網址有 24 小時的有效期。超大檔案也有充裕的時間完成上傳
